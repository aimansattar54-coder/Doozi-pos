Build **Doozi POS**, a polished, enterprise-grade point-of-sale + business-management web app for a Maldives retail store (currency **MVR**, **GST** tax). Primarily a **static front-end** (HTML/CSS/JS, no framework) that runs fully offline on `localStorage`/IndexedDB, **plus an OPTIONAL local backup server** for real disk-backed backup/restore + auto-sync (see §8). ## 3. KEY DECISIONS (do not redo)
- **IDs are strings** (`P001`, `TXN-1011`, `C001`…). Never `parseInt` an ID; always quote them in inline `onclick` (`fn('${x.id}')`).
- **Variants:** variant ids are strings like `P011-V1` (or `P011-V<base36>` for ones added via the editor). A cart line's `id` may be a **product id OR a variant id** — always resolve through `DB.sellable(id)` / `DB.applyStockDelta(id,…)`, never assume `it.id` is a product. Parent `stock` is **derived** from variants (don't write it directly for variant products; `syncParentStock` mirrors it).
- **Money:** `DB.money()` rounds to cents; **refunds are excluded from revenue everywhere**. `DB.addTransaction` is the single source of truth for stock decrement + CRM; `DB.reverseTransaction` handles refunds (guarded against double-refund).
- `DB.normalize()` runs on load: backfills schema + **resyncs all ID counters** from data (no collisions).
- **THEME IS TEAL — never change colors unless explicitly told.** "Change the design" = layout/structure, *not* palette. (Recoloring to purple once made the user upset.) Keep **footer** and **login page** teal regardless.
- **Products use a letter-badge `emoji`** (first letter of name), not real emojis — user asked to remove all product emojis.
- **Persistence:** `DB.save()` is hooked into `navigate()`; ads use IndexedDB (not localStorage) so any video size persists.
- **`.mov` videos** are relabeled to `video/mp4` so Chrome plays them.
- Dropdowns deliberately use the **login teal** color.

## 8. CLOUD BACKUP SERVER (optional, local)
**Goal:** give the app a real place to back up to + auto-sync, without a hosted account. Works on this Node-less Windows machine via a PowerShell server; a Node version is also included for portability.

**Files**
- `server.js` — **Node** server: serves the app *and* the API on one port (same-origin, no CORS). Use this if Node is available (it isn't on this machine).
- `backup-server.ps1` — **PowerShell** server (HttpListener on `http://localhost:3590/`), zero dependencies. This is what runs here. CORS-enabled so the statically-served app (port 3400) can call it.
- `js/cloud.js` — client sync. **Auto-detects** the backend: tries same-origin `/api/health` first (Node case), then `http://localhost:3590` (PowerShell case); remembers the winner in `Cloud._base`. Degrades silently to local-only when neither is up.

**API:** `GET /api/health`, `GET /api/latest` (204 if empty), `GET /api/backups`, `POST /api/backup`. Backups are JSON files under `./backups/` — `latest.json` (live copy) + throttled `snap-<ts>.json` history (≥5 min apart, newest 50 kept).

**Sync model (auto-sync, last-write-wins):** `DB._meta.updatedAt` is the clock. `DB.save()` bumps it and calls `Cloud.schedulePush()` (debounced 1.5 s). On load, `Cloud.init()` resolves the base, then `pull()`s: applies the server copy only if `remoteAt > localAt`, else pushes local up.
- **Critical ordering fix:** `DB._loadedUpdatedAt` (captured at the bottom of `data.js` right after load) is the comparison baseline — NOT the live `_meta.updatedAt`, which startup saves bump. Auto-pushes are **gated by `Cloud._ready`** until the first pull finishes, so a fresh seed can't clobber a populated server. `_loadedUpdatedAt` is re-synced after every successful push/pull.

**To run the backup server (PowerShell):**
```powershell
cd "C:\Users\aiman\Desktop\claude\POS X"
powershell -ExecutionPolicy Bypass -File backup-server.ps1
# leave it open; Settings → Backup & Sync should show "Synced"
```
**Settings → Backup & Sync:** status badge, auto-sync on/off (`doozi_cloud_off` flag), Back up now / Sync from cloud / Refresh history, backup-history list, and the "About cloud features" note (multi-user/internet-sync/audit still need hosting).

**Gotchas:** the Claude **preview** runner serves static files via its own `serve.ps1` (it does NOT run `server.js`; `node` isn't installed) — so in preview the app is on :3400 and the PowerShell API is the reachable backend on :3590. Don't point `launch.json` at `node`. To fully reset: stop the server, delete `backups/*.json`, clear `localStorage`, reload (the client re-seeds the empty server via the 204 path).
